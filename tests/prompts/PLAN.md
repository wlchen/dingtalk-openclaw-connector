# 提示词模块（buildMediaSystemPrompt）测试方案

## 1. 模块划分与职责

- **buildMediaSystemPrompt(): string**
  - 职责：返回钉钉渠道下发给 AI 的系统提示词，约定图片/视频/音频/文件的展示与分享规则，确保 AI 输出格式与插件解析一致（如 DINGTALK_VIDEO、DINGTALK_AUDIO、DINGTALK_FILE、本地路径写法等）。

## 2. 覆盖要点（全量检查项）

### 2.1 必须包含的章节标题

| 序号 | 必须包含的文本 | 说明 |
|------|----------------|------|
| 1 | `## 钉钉图片和文件显示规则` | 主标题 |
| 2 | `### 一、图片显示` | 图片章节 |
| 3 | `### 二、视频分享` | 视频章节 |
| 4 | `### 三、音频分享` | 音频章节 |
| 5 | `### 四、文件分享` | 文件章节 |

### 2.2 图片显示规则

| 序号 | 必须包含的文本/关键词 | 说明 |
|------|------------------------|------|
| 6 | `file://` | 正确方式示例 |
| 7 | `/tmp/` 或 `/Users/` | 本地路径示例 |
| 8 | `不要自己执行 curl 上传` | 禁止行为 |
| 9 | `不要对路径进行转义` 或 `反斜杠` | 禁止转义路径 |
| 10 | `系统会自动上传到钉钉` | 行为说明 |

### 2.3 视频分享规则

| 序号 | 必须包含的文本/关键词 | 说明 |
|------|------------------------|------|
| 11 | `[DINGTALK_VIDEO]` | 视频标记格式 |
| 12 | `[/DINGTALK_VIDEO]` | 闭合标签 |
| 13 | `"path"` | 参数名 |
| 14 | `mp4` | 支持格式 |
| 15 | `20MB` | 大小限制 |
| 16 | `时长、分辨率` 或 `封面` | 元数据说明 |

### 2.4 音频分享规则

| 序号 | 必须包含的文本/关键词 | 说明 |
|------|------------------------|------|
| 17 | `[DINGTALK_AUDIO]` | 音频标记格式 |
| 18 | `[/DINGTALK_AUDIO]` | 闭合标签 |
| 19 | `ogg` 或 `amr` | 支持格式 |
| 20 | `20MB` | 大小限制 |
| 21 | `音频时长` | 行为说明 |

### 2.5 文件分享规则

| 序号 | 必须包含的文本/关键词 | 说明 |
|------|------------------------|------|
| 22 | `[DINGTALK_FILE]` | 文件标记格式 |
| 23 | `[/DINGTALK_FILE]` | 闭合标签 |
| 24 | `fileName` 或 `文件名` | 参数说明 |
| 25 | `fileType` 或 `扩展名` | 参数说明 |
| 26 | `20MB` 或 `文件过大` | 大小限制 |

### 2.6 稳定性与一致性

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 27 | 多次调用返回值全等 | 无随机或时间戳，纯静态字符串 |
| 28 | 返回类型为 string 且长度 > 0 | 非空 |

## 3. 预期正确输出与潜在错误

- **正确输出**：单次调用返回完整 Markdown 段落，包含上述所有章节与关键词，且多次调用结果一致。
- **潜在错误原因**：漏写某一章节导致 AI 未遵守对应格式；标记名拼写错误（如 DINGTALK_VIDOE）；20MB 写成 20 Mb 或 20M 导致歧义；示例 JSON 中 path/fileName/fileType 缺一导致 AI 少填字段。
